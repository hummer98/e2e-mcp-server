

このプロジェクトはClaudeCodeなどのソフトウェアエンジニアAIエージェントにおいて、E2Eテストを円滑に実行させるためのModel Context Protocolサーバーを提供します。
従来のplaywright mcpはサーバー管理機能を持たないためplaywrightがアクセスすべき開発サーバーの状態管理をClaudeCode自身が行う必要があり、非常に煩雑でした。この問題を解決するためサーバーのプロセス管理をMCPサーバーに移譲することでE2Eテスト実行に専念できるようにするのが狙いです。

# このMCPサーバーの主な機能
- E2Eセッションの開始: AIにセッション開始を要求されたとき、**サーバー起動コマンド**を叩いてサーバーを起動させます。この起動コマンドの標準出力にはアクセスすべきURLを返すように期待します。MCPサーバーのレスポンスとしてはこのURLを返し、サーバーがE2Eテスト可能状態であることを知らせます
- E2Eセッションの終了: **サーバー起動コマンド**を`--shutdown`オプションで呼び出します。
- playwright機能: すべてのplaywright機能を提供します。

## サーバー起動コマンドの仕様

ユーザーが提供するサーバー起動コマンドには以下の仕様を要求します
`(オプションなし)`or`--start`: サーバーを起動し、規定のJSONを返却します
`--restart`: 強制的にリスタートするコマンドです
`--shutdown`: サーバーをシャットダウンさせます
`--status`: サーバーの起動状況を返します。 ### statusレスポンス
